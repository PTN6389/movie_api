myFlix API Documentation

REST API for the myFlix application that interacts with a database that stores data about different movies

Business Logic URL Endpoint HTTP Method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None JSON object listing details about all movies
Return data about a single movie by title to the user /movies/[title] GET None JSON object with details about a single movie (description, genre, director, image URL, whether it's featured or not)
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[title]/genre GET None Text message with the genre for a movie title
Return data about a director by name /movies/[director] GET None JSON object with details about a director (bio, birth year, death year)
Allow new users to register /users POST JSON object with details about the user JSON object with details about the added user including ID
Allow users to update their user info (username) /users/[user]/[username] PUT None Text message with confirmation that the username was updated
Allow users to add a movie to their list of favorites /users/[username]/[title] POST None (showing only a text that a movie has been added—more on this later)
Allow users to remove a movie from their list of favorites /users/[username]/[title] DELETE None (showing only a text that a movie has been removed—more on this later)
Allow existing users to deregister /users/[username] DELETE None (showing only a text that a user email has been removed—more on this later)
